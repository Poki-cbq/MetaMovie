# MovieInsight — 需求文档

> 全栈电影数据发现与分析平台  
> 版本：v1.0 MVP  
> 审核状态：⏳ 待审核

---

## 一、项目定义

### 1.1 一句话描述

基于 TMDB 免费 API 的全栈电影数据平台——用户可以浏览/筛选/搜索热门电影，查看数据分析看板。

### 1.2 项目名称

**MovieInsight**（电影洞察）

### 1.3 目标用户

- **电影爱好者**：系统性地发现和筛选电影
- **面试官/招聘方**：10 秒理解项目功能，然后看代码质量
- **GitHub 开源社区**：clone 即跑，零配置

### 1.4 核心价值主张

| 用户痛点 | MovieInsight 解决方式 |
|----------|----------------------|
| 选择困难，不知道看什么电影 | 按类型/年份/评分/热度多维度筛选 |
| 只看评分不够，想了解数据规律 | 可视化看板：评分分布、年代趋势、类型占比 |
| 看到好电影想了解导演/演员 | 详情页演职表，后续 V2 加入电影人时间线 |

## 三、数据需求

### 3.1 数据来源

**TMDB API v3**（The Movie Database）
- 免费注册获取 API Key：https://www.themoviedb.org/settings/api
- 速率限制：每分钟 40 次请求
- 返回语言：`language=zh-CN`（中文标题+简介）+ 保留 `original_title`（英文原名）

### 3.2 种子数据量

- 流行电影 8 页 × 20 = 160 条
- 高分电影 5 页 × 20 = 100 条
- 去重后约 **200-260 部电影**

### 3.3 数据字段

每条电影存储以下字段：

| 字段 | 类型 | 来源 | 说明 |
|------|------|------|------|
| tmdb_id | int | API | TMDB 唯一标识 |
| title | str | API（zh-CN） | 中文标题 |
| original_title | str | API | 原始标题（英文） |
| overview | str | API（zh-CN） | 中文剧情简介 |
| poster_path | str | API | 海报路径，前端拼接 CDN URL |
| backdrop_path | str | API | 背景图路径 |
| release_date | str | API | 上映日期 YYYY-MM-DD |
| runtime | int | API | 片长（分钟） |
| vote_average | float | API | TMDB 评分（0-10） |
| vote_count | int | API | 评分人数 |
| popularity | float | API | 热度指标 |
| budget | int | API | 预算（美元） |
| revenue | int | API | 票房（美元） |
| original_language | str | API | 原始语言代码 |
| tagline | str | API | 电影标语 |
| genres | str | API → 转中文 | 类型，逗号分隔（如"动作,犯罪,剧情"） |
| production_countries | str | API | 制片国家，逗号分隔（如"美国,英国"） |

### 3.4 演职人员数据

| 字段 | 说明 |
|------|------|
| person_id | TMDB 人物 ID |
| name | 中/英文名 |
| profile_path | 头像路径 |
| role | 演员时 = 角色名 / crew 时 = 职位名（Director 等） |
| credit_type | "cast"（演员）或 "crew"（工作人员） |
| department | crew 的部门（Directing / Writing） |
| order | 排序（演员按重要性，crew 按职位） |

### 3.5 海报处理

- 数据库只存 `poster_path`，不下载图片到本地
- 前端拼接 CDN URL：`https://image.tmdb.org/t/p/w500{poster_path}`
- 加载失败时显示骨架屏动画

## 四、UI/UX 需求

### 4.1 整体布局

- **侧边栏导航**：左侧竖排菜单，200px 宽
- **完整页脚**：TMDB 数据来源标注 + GitHub 链接
- **桌面端**：适配 1920px / 1440px / 1366px

### 4.2 配色方案

| 用途 | 颜色 | 说明 |
|------|------|------|
| 导航/页头 | `#0d253f` | TMDB 深蓝 |
| 强调/链接 | `#01b4e4` | TMDB 浅蓝 |
| 高分 | `#21d07a` | ≥7.5 分高亮绿 |
| 中分 | `#d2d531` | 5-7.5 分黄色 |
| 低分 | `#db2360` | <5 分红色 |
| 看板背景 | 暗色 | 数据分析页专用 |
| 其他页面背景 | 亮色 `#f5f5f5` | 与 Element Plus 默认主题一致 |

### 4.3 发现页

- 4 列卡片网格
- 每张卡片：海报（w500）+ 标题 + 5星评分 + 类型标签
- 顶部筛选栏：搜索框 + 类型下拉 + 年份范围滑块 + 国家下拉 + 排序下拉
- 底部分页器
- 海报懒加载
- 海报加载中显示骨架屏动画

### 4.4 详情页

- 新路由跳转 `/movie/:id`
- 左侧海报大图（w500），右侧信息区
- 信息区：标题（中+英）+ 5星评分 + 评分人数 + 片长(2h32min) + 类型标签 + 国家 + 语言
- 预算/票房：缩写格式（$185M / $1.0B）
- 剧情简介
- 演职表：Element Plus 表格，导演 + Top 10 演员，头像+名字+角色
- 导演/演员名不可点击（V2 才做电影人页）

### 4.5 数据分析页

- 3 张 ECharts 图表，2×2 网格布局（第 4 格留空或放说明）
- 图表 1：评分分布直方图（bar）
- 图表 2：类型分布饼图（pie）
- 图表 3：年代趋势折线图（line，每年电影数量）
- 图表使用暗色主题，与看板页整体风格统一

### 4.6 空状态 & 错误处理

- 数据库为空：文字提示"暂无数据，请先运行 python seed.py 初始化数据"
- API 请求失败：Element Plus ElMessage 顶部错误提示
- 海报加载失败：卡片显示骨架屏占位

## 五、非功能需求

| 类别 | 要求 |
|------|------|
| 安装复杂度 | `pip install -r requirements.txt` + `npm install`，零配置启动 |
| 数据库 | SQLite，自动建表，无需手动安装数据库服务 |
| 性能 | 300 条数据下所有 API 响应 < 200ms |
| 浏览器兼容 | Chrome / Edge / Firefox 最新版 |
| 国际化 | 界面中文，电影数据中英双语（title + original_title） |
| 代码注释 | 中文注释 |
| 测试 | pytest API 集成测试，覆盖所有端点 |
